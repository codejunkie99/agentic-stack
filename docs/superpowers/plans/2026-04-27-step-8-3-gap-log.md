# Step 8.3 — Gap Log

> Running log of gaps surfaced during the dry-run. One entry per gap.
> Severity: blocker / annoying / nice-to-have. Status:
> open / fix-on-this-branch / deferred-to-8.4 / deferred-to-8.5.

## Gap 1 — Install does not refresh `.agent/` brain on reinstall

- **Severity:** annoying (blocker for tight harness-dev loop, not for users)
- **Surfaced:** Stage 2 verification — sandbox install
- **Symptom:** New `.agent/tools/*.py` and skills added in fork did not
  reach `examples/pdlc-sandbox/` on reinstall. Required manual `cp` to
  test. Same applies to fixes in `.agent/AGENTS.md`, `.agent/protocols/`,
  any updated tool.
- **Root cause:** [harness_manager/install.py:189](../../../harness_manager/install.py)
  copies `.agent/` only when target's `.agent/` does not exist. Once the
  brain is dropped, subsequent installs only refresh adapter-specific
  files listed in `adapter.json`. The "in-place evolution" intent
  (Philosophy A) is correctly preserved for memory + skills, but
  fork-managed tools/protocols/AGENTS.md should refresh.
- **Proposed fix (deferred):** add `--upgrade` flag to install that
  refreshes specific subdirs (`.agent/tools/`, `.agent/protocols/`,
  `.agent/harness/`, `.agent/AGENTS.md`) but never touches
  `.agent/memory/`, `.agent/skills/` (in `in_place` mode), or
  `.claude/agent-memory/`.
- **Status:** deferred-to-8.5 (this is exactly the
  `install.sh --upgrade` work that was already scoped there).
- **Workaround:** manual `cp` for sandbox iteration; fresh installs
  (Stage 3 target at `~/code/case-dryrun-<slug>/`) avoid the issue.

## Gap 2 — `take_install_snapshot` constructed doubled path

- **Severity:** blocker (snapshot capture failed silently in install output)
- **Surfaced:** Stage 2 verification — sandbox install
- **Symptom:** `take_install_snapshot: snapshot_failed —
  ...examples/pdlc-sandbox/examples/pdlc-sandbox/.agent/tools/
  snapshot_diff.py: [Errno 2] No such file or directory`
- **Root cause:** subprocess passed a relative tool path AND
  `cwd=target_root`, so Python re-resolved the path against cwd.
- **Status:** fixed-on-this-branch, commit `1125255`.

## Gap 3 — `document-researcher` ran into PDF + image limits

- **Severity:** annoying (workaround exists; degrades summary quality
  for non-text formats)
- **Surfaced:** Stage 3c — processing 8 uploads
- **Symptom:** `pdftotext` and `pdftoppm` (poppler) not available in
  the local environment; PNG (image) summaries had no auto-extraction
  path. Both were handled defensively (PDF deferred to its `.md`
  sibling; PNG marked duplicate-of-md).
- **Root cause:** `document-researcher` skill assumes UTF-8 readability;
  needs a graceful-degradation rule for binary formats.
- **Proposed fix:** update `document-researcher/SKILL.md` to require
  a fallback summary even when content can't be auto-extracted —
  user-description goes in, file metadata + topics go in, INDEX entry
  is created with `topics: image-only` or `topics: pdf-no-extractor`
  flag. Also document the optional dependencies (`pdftotext`,
  `pdftoppm`, `python-pptx`) in the skill's preconditions.
- **Status:** open (would land as `skill-update: document-researcher`
  via the per-skill self-rewrite hook the skill already documents).

## Gap 4 — sandbox install reinstall blocked by missing install.json

- **Severity:** annoying (reproducibility concern for harness dev loop)
- **Surfaced:** Stage 2 verification — first sandbox reinstall attempt
- **Symptom:** `error: examples/pdlc-sandbox/.agent/ exists but
  install.json does not. this looks like a pre-v0.9 install.` Doctor
  command refused to register in non-interactive shell; required
  manually writing a stub install.json to proceed.
- **Root cause:** doctor flow assumes interactive terminal; no `--yes`
  shortcut for the register-pre-v0.9-adapter case.
- **Proposed fix:** extend `cmd_doctor` to honor `--yes` for the
  register-existing-adapters path, OR have install.sh write a minimal
  install.json stub when it sees the "brain present, no install.json"
  state and `--yes` is set.
- **Status:** open, candidate for Step 8.4 or 8.5.

## Gap 6 — `document-researcher` requires `pypdf` for PDF inputs

- **Severity:** annoying → resolved (`pip install --user pypdf`)
- **Surfaced:** Stage 3c re-run with new files (April 28); 5 of 7 new
  files were PDFs that would have been deferred-to-on-demand under
  the v1 skill rules.
- **Symptom:** With pypdf installed, full-text extraction was clean
  for HarnessX_v4 (28 pages), BOCHK (37 pages), Vince transcript
  (13 pages), Balaji (31 pages), SC AI SDLC (102 pages). Without
  pypdf, all of these would have fallen to "deferred-to-md-sibling"
  treatment — useless for the BOCHK + Balaji decks which had no
  markdown twins.
- **Proposed fix:** add `pypdf` (pure Python, no system deps) to the
  `document-researcher` skill's optional-tools list and document a
  one-liner setup in the skill preconditions: `pip install --user
  pypdf` if PDFs are expected. Better than poppler/pdftotext (system-
  dep, requires brew install).
- **Status:** open as skill-update candidate to document-researcher.

## Gap 7 — Adding a new skill mid-engagement requires manual sync to target

- **Severity:** blocker for tight skill iteration; same root cause as Gap 1
- **Surfaced:** Stage 16 (consulting-deck-builder skill build mid-dry-run)
- **Symptom:** Built `consulting-deck-builder/SKILL.md` in fork on
  feature branch. Reinstall to target via `./install.sh claude-code
  <target>` did NOT propagate the new skill — `.agent/skills/` only
  copies on first install (per Gap 1). Required manual `cp -r` of
  the new skill dir + manual sync of `_index.md` + `_manifest.jsonl`
  to target.
- **Reproducer:** create new skill in fork, reinstall to existing
  target, observe target's `.agent/skills/` is unchanged.
- **Proposed fix:** the deferred `install.sh --upgrade` (Gap 1 fix in
  Step 8.5) should explicitly cover skill propagation. Specifically:
  refresh `.agent/skills/_index.md` + `_manifest.jsonl` always
  (cheap), and copy missing skill subdirs (skip existing ones in
  `in_place` mode unless user passes `--upgrade-skills`).
- **Status:** open, escalates priority of the 8.5 `--upgrade` work.

## Gap 5 — `permissions.deny` block correctly fires against installer

- **Severity:** false alarm (turned out to be macOS bash sandbox, not
  a deny-rule misfire)
- **Surfaced:** Stage 2 verification — first install after deny block landed
- **Symptom:** `PermissionError: [Errno 1] Operation not permitted:
  'examples/pdlc-sandbox/.claude/agents/architect.md'` during install.py
- **Resolution:** the deny rules don't apply to Python file writes from
  outside the Claude Code session. The error was the bash sandbox
  blocking writes outside its allowlist. Retried with
  `dangerouslyDisableSandbox: true` and succeeded.
- **Status:** not-a-bug. Documented because the symptom looked like a
  deny-rule problem on first read.
