# Case Engagement Process

## Engagement Phases

### Phase 1: Discovery (Weeks 1-2)
- Objectives: Align on problem, collect baseline data, conduct stakeholder interviews
- Key Deliverables: Project Charter, Situation Assessment, Data Collection Plan
- Gate: Steering committee sign-off on problem statement and approach

### Phase 2: Analysis (Weeks 3-6)
- Objectives: Execute analytical workplan, develop hypotheses, test against data
- Key Deliverables: Issue Tree, workstream analyses
- Gate: Mid-case review deck presented to client leadership

### Phase 3: Synthesis & Recommendation (Weeks 7-9)
- Objectives: Synthesize findings, develop recommendations, pressure-test with client
- Key Deliverables: Preliminary recommendations, client workshop
- Gate: Client alignment on recommendation direction

### Phase 4: Delivery (Week 10+)
- Objectives: Finalize and deliver recommendations, handoff to client implementation team
- Key Deliverables: Final recommendations deck, implementation roadmap
- Gate: Final sign-off by partner and client sponsor

## Key Meeting Cadences

| Meeting | Frequency | Participants | Purpose |
|---|---|---|---|
| Steering Committee | Weekly | Full team + client leadership | Status, decisions, escalations |
| Working Sessions | 2-3x/week | BCG team + client counterparts | Analysis progress, data access |
| Internal Team Sync | Daily | BCG team only | Coordination, blockers |
| Partner Review | Weekly | BCG team only | Quality review of outputs |

## Document Standards
- All documents versioned as vN (v1, v2, v3...)
- Working drafts flagged as [DRAFT] in title
- Final versions approved by Partner before sharing with client
- All documents stored in Confluence under the case space

## Decision Rights

| Decision Type | Owner |
|---|---|
| Scope changes | Partner + Client Sponsor |
| Analytical approach | Project Leader |
| Recommendation direction | Partner |
| Day-to-day task prioritization | Project Leader |
| Data access requests | Client Counterpart |
