## Purpose
Transform a case question into a MECE issue tree with a supporting hypothesis network that guides the analytical agenda.

## Contents
1. Root Question (the overarching case question)
2. Issue Tree (MECE decomposition, 2-3 levels)
3. Hypothesis Network (one hypothesis per issue tree node)
4. Evidence Map (what data would prove/disprove each hypothesis)
5. Prioritization (which branches are highest priority and why)
6. Analytical Work Package List (what analyses to run and in what sequence)

## Team Structure: Flat
- **framework-lead** — Owns end-to-end structure
- **case-analyst** — Provides analytical grounding for each branch

## Review
- **partner-analytics** — Analytical rigor and completeness

## Output Format
Markdown with structured hierarchy. Include mermaid diagram of issue tree if helpful.

## Quality Gates
- MECE test passes (verified at each level)
- Every leaf node maps to a specific analytical work package
- Hypotheses are falsifiable directional claims
